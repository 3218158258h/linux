
## 1. 自旋锁是什么
- 一种**忙等锁**：抢不到锁时，不阻塞、不睡眠，**循环 CAS 尝试获取锁**
- 适合：**临界区极短、不阻塞、不睡眠**的场景
- 优点：**无上下文切换开销**，竞争小时极快
- 缺点：竞争大时**空转浪费 CPU**

---

## 2. 内核态自旋锁（真正靠谱的自旋锁）
- 用于：**Linux 内核、驱动、中断上下文**
- 特点：
  1. 持有锁时**关闭内核抢占**
  2. 持锁线程**不会被其他线程强行切走**
  3. 保证快速执行、快速释放
  4. 其他线程自旋时间很短，不会白转很久
- 规则：
  - 临界区**必须极短**
  - **绝对不能睡眠、阻塞、调度**
  - 否则会导致其他 CPU 长时间自旋，系统卡顿

---

## 3. 用户态自旋锁（pthread_spinlock）
- 线程是**内核级线程（pthread）**，但运行在**用户态**
- 用户态**没有权限关闭抢占/调度**
- 问题：
  1. 持锁线程**随时可能因时间片耗尽被切走**
  2. 其他线程会**长时间空转自旋**
  3. 单 CPU 下极易造成**等效死锁**
  4. CPU 利用率飙升，性能极差
- 结论：
  **用户态业务代码几乎不应该使用自旋锁**

---

## 4. 内核态 vs 用户态自旋锁的核心区别
- 内核态：**能关抢占 → 持锁线程不被切走 → 自旋高效安全**
- 用户态：**无权关抢占 → 持锁线程随时被切走 → 自旋空转浪费CPU**

---

## 5. 自旋锁失效/不能用的场景
1. 临界区**太长**
2. 临界区**可能睡眠/阻塞**
3. **单 CPU** 环境
4. 用户态 pthread 直接使用
5. 锁嵌套顺序不一致导致**死锁**
6. 中断上下文与线程混用同一把锁
