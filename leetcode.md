## 一、哈希
### 1. 两数之和【简单】
**题目描述**  
给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值 `target` 的那两个整数，并返回它们的数组下标。  
你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。你可以按任意顺序返回答案。
**解题思路**  
用哈希表（Map）记录每个数值对应的下标。遍历数组，对于当前元素 `x`，在哈希表中查找 `target - x` 是否存在。如果存在，直接返回两个下标；如果不存在，将当前值和下标存入哈希表。这样只需要一次遍历，时间复杂度 O(n)，空间复杂度 O(n)。

### 2. 49. 字母异位词分组【中等】
**题目描述**  
给你一个字符串数组 `strs`，请你将「字母异位词」分组在一起。可以按任意顺序返回结果列表。  
字母异位词：由重新排列源单词的所有字母得到的一个新单词（例如 `eat`、`ate`、`tea` 互为字母异位词）。
**解题思路**  
将每个字符串按字母排序后作为“标准键”，所有字母异位词排序后的结果相同，因此可以将它们映射到同一个列表。例如 `"eat"`、`"ate"` 都排序为 `"aet"`，放入同一个分组。也可以用长度为 26 的计数数组作为键，避免排序，但代码稍复杂。整体时间复杂度约为 O(NKlogK)，N 是字符串数量，K 是字符串平均长度。

### 3. 128. 最长连续序列【中等】
**题目描述**  
给定一个未排序的整数数组 `nums`，找出数字连续的最长序列的长度。要求算法的时间复杂度为 O(n)。
**解题思路**  
先把所有数字放入哈希集合，实现 O(1) 查找。对集合中的每个数 `x`，只有当 `x-1` 不在集合中时，才把它当作一个连续序列的起点，然后不断尝试 `x+1, x+2...` 是否存在，统计当前序列长度。这样每个序列最多被完整遍历一次，整体时间复杂度 O(n)。

## 二、双指针
### 4. 283. 移动零【简单】
**题目描述**  
给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。必须在不复制数组的情况下原地对数组进行操作。
**解题思路**  
使用快慢指针。慢指针 `slow` 记录当前已经处理好的非零元素末尾，快指针 `fast` 遍历整个数组。当 `fast` 遇到非零元素时，将其交换到 `slow` 的位置，然后 `slow` 向后移一位。这样所有非零元素都被依次移到前面，剩下的位置自然就是 0。时间复杂度 O(n)，空间复杂度 O(1)。

### 5. 11. 盛最多水的容器【中等】
**题目描述**  
给定一个长度为 `n` 的整数数组 `height`，其中 `height[i]` 表示在第 `i` 点的高度。两条垂直线 `x = i` 和 `x = j` 之间能容纳的最大水量为：  
`min(height[i], height[j]) * (j - i)`。  
请你找出使容器盛最多水的两条线，返回最大容量。
**解题思路**  
使用左右双指针，初始在数组两端。容量由较短边和两指针距离决定。每次将较短边的指针向中间移动，才有可能获得更大的容量（如果移动较长边，距离变小，高度仍受较短边限制，容量只会变小）。每次移动后计算当前容量并更新最大值。时间复杂度 O(n)，空间复杂度 O(1)。

### 6. 15. 三数之和【中等】
**题目描述**  
给定一个整数数组 `nums`，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j, i != k, j != k` 且 `nums[i] + nums[j] + nums[k] == 0`。请你返回所有不重复的三元组。答案中不可以包含重复的三元组。
**解题思路**  
先对数组排序。固定一个指针 `i`，然后在剩下的区间使用双指针 `left` 和 `right`，寻找两数之和等于 `-nums[i]`。如果和偏小，`left` 右移；如果和偏大，`right` 左移。找到解后要跳过重复元素以避免重复结果。时间复杂度 O(n²)，空间复杂度主要是排序的 O(logn)。

### 7. 42. 接雨水【困难】
**题目描述**  
给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。
**解题思路**  
双指针法最优。维护 `left`、`right` 指针以及 `leftMax`、`rightMax` 记录左右两边的最高柱子。对于较低的一边，当前柱子能接的雨水量由该边的最大高度减去当前高度决定（因为另一边一定有更高的柱子挡住水）。移动较低指针并更新最大值，累加雨水量。时间复杂度 O(n)，空间复杂度 O(1)。

### 8. 3. 无重复字符的最长子串【中等】
**题目描述**  
给定一个字符串 `s`，请你找出其中不含有重复字符的最长子串的长度。子串是字符串中连续的字符序列。
**解题思路**  
使用滑动窗口 + 哈希集合。维护左右指针 `left`、`right`，窗口内的字符存入集合。`right` 不断向右扩展，遇到重复字符时，`left` 向右移动直到窗口内不再包含该字符。每次成功扩展后更新最大长度。时间复杂度 O(n)，每个字符最多被访问两次。

### 9. 438. 找到字符串中所有字母异位词【中等】
**题目描述**  
给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的字母异位词的起始索引。返回一个包含所有起始索引的列表。  
字母异位词：由相同字母重新排列形成的字符串，例如 `p = "abc"`，则 `s` 中的 `"abc"`、`"bca"` 等都是字母异位词。
**解题思路**  
使用固定大小的滑动窗口。先统计 `p` 中每个字母的出现次数，维护一个长度为 `len(p)` 的窗口，统计窗口内各字母计数。当窗口内计数与 `p` 完全一致时，记录窗口起始索引。每次窗口右移，只需要更新最左和最右两个字符的计数。时间复杂度 O(|s|+|p|)。

### 10. 560. 和为 K 的子数组【中等】
**题目描述**  
给定一个整数数组 `nums` 和一个整数 `k`，请你统计并返回数组中连续子数组的和等于 `k` 的个数。
**解题思路**  
前缀和 + 哈希表。定义前缀和 `pre[i]` 为 `nums[0..i]` 的和。对于每个位置 `i`，我们需要统计有多少个 `j < i` 满足 `pre[i] - pre[j] = k`，即 `pre[j] = pre[i] - k`。用哈希表记录每个前缀和出现的次数，一边遍历一边统计。初始化时令 `pre[0] = 0` 出现 1 次。时间复杂度 O(n)。

### 11. 239. 滑动窗口最大值【困难】
**题目描述**  
给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组的最左侧移动到最右侧。你只能看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。返回滑动窗口中的最大值构成的数组。
**解题思路**  
使用单调队列（双端队列）存储下标，保持队首到队尾的下标对应的元素单调递减。遍历数组，当队首下标超出窗口范围时，弹出队首。每次将当前下标加入队列前，弹出所有比当前元素小的队尾元素，保证队列单调性。此时队首下标对应的元素就是当前窗口最大值。时间复杂度 O(n)。

### 12. 76. 最小覆盖子串【困难】
**题目描述**  
给你一个字符串 `s` 和一个字符串 `t`，返回 `s` 中涵盖 `t` 所有字符（包括重复字符）的最小子串。如果 `s` 中不存在这样的子串，则返回空字符串。  
注意：如果 `s` 中存在这样的子串，我们保证它是唯一的答案。
**解题思路**  
滑动窗口。用哈希表记录 `t` 中每个字符的需求次数，再用一个哈希表记录当前窗口中的字符计数。维护一个变量 `needCnt` 表示还需要满足的字符总数。扩展右边界直到窗口满足条件，然后收缩左边界寻找最小窗口。每次找到满足条件的窗口时更新最小长度和起始位置。时间复杂度大致 O(|s|+|t|)。

### 13. 53. 最大子数组和【中等】
**题目描述**  
给你一个整数数组 `nums`，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。
**解题思路**  
动态规划或贪心。设 `dp[i]` 表示以 `nums[i]` 结尾的最大子数组和，则 `dp[i] = max(nums[i], dp[i-1] + nums[i])`。最后答案是所有 `dp[i]` 中的最大值。可以优化为只保留一个变量 `cur` 和一个 `ans`。时间复杂度 O(n)，空间复杂度 O(1)。

### 14. 56. 合并区间【中等】
**题目描述**  
以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]`。请你合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。
**解题思路**  
先按区间左端点排序。遍历区间，如果当前区间的左端点小于等于结果中最后一个区间的右端点，则合并（更新右端点为两者较大值）；否则直接加入结果。排序保证了合并的有序性。时间复杂度 O(nlogn)，主要是排序。

### 15. 189. 轮转数组【中等】
**题目描述**  
给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。
**解题思路**  
经典三次翻转法：  
1. 先翻转整个数组；  
2. 再翻转前 `k % n` 个元素；  
3. 最后翻转剩下的元素。  
例如 `[1,2,3,4,5,6,7]`，k=3，整体翻转为 `[7,6,5,4,3,2,1]`，前 3 翻转为 `[5,6,7,4,3,2,1]`，后 4 翻转为 `[5,6,7,1,2,3,4]`。时间复杂度 O(n)，空间复杂度 O(1)。

### 16. 238. 除自身以外数组的乘积【中等】
**题目描述**  
给定一个整数数组 `nums`，返回数组 `answer`，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积。题目数据保证数组中任意元素的全部前缀元素和后缀（甚至整个数组）的乘积都在 32 位整数范围内。  
请不要使用除法，且在 O(n) 时间复杂度内完成。
**解题思路**  
分别计算前缀乘积和后缀乘积 。对于每个位置 `i`，`answer[i] = pre[i] * suf[i]`，其中 `pre[i]` 表示 `i` 左侧所有元素的乘积，`suf[i]` 表示右侧所有元素的乘积。
1. 计算前缀积
初始化`结果数组` answer，answer[0] = 1（第一个元素左侧无元素，乘积为 1）。
从`左到右`遍历数组：answer[i] = answer[i-1] × nums[i-1]，此时 answer[i] 存储 i 左侧所有元素的乘积。
2. 计算后缀积并合并
初始化`后缀积变量` right = 1（最后一个元素右侧无元素，乘积为 1）。
从`右到左`遍历数组：answer[i] = answer[i] × right，同时更新 right = right × nums[i]，将后缀积累乘到结果中。
时间复杂度 O(n)，空间复杂度 O(1)（输出数组不算额外空间）。
```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        vector<int> answer(nums.size(),1);//结果数组
        int right = 1;//后缀乘积

        //第一次遍历，存储前缀积
        for(int i = 1;i<nums.size();i++)
        {
            answer[i] = answer[i-1]*nums[i-1];
        }
        //第二次遍历，使用后缀积乘前缀积得到结果
        for(int i = nums.size()-1;i>=0;i--)
        {
            answer[i] = right*answer[i];
            right = right*nums[i];
        }
        //时间复杂度O(2n)=O(n)
        return answer;
    }
};
```

**双指针法**
```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        int lsum = 1, rsum = 1;
        vector<int> res(n, 1);
        for (int l = 0, r = n - 1; l < n; l++, r--) {
            res[l] *= lsum, res[r] *= rsum;
            lsum *= nums[l], rsum *= nums[r];
        }
        return res;
    }
};
```

### 17. 41. 缺失的第一个正数【困难】
**题目描述**  
给定一个未排序的整数数组 `nums`，请你找出其中没有出现的最小的正整数。要求算法的时间复杂度为 O(n)，并且只能使用常数级别的额外空间。
**解题思路**  
将数组本身当作哈希表。遍历数组，对于每个在 `1..n` 范围内的数 `x`，尝试把它放到下标 `x-1` 的位置上（交换）。放置完后，再遍历一次，第一个不满足 `nums[i] == i+1` 的位置 `i`，缺失的正数就是 `i+1`。如果全部满足，则缺失的是 `n+1`。时间复杂度 O(n)，空间复杂度 O(1)。

### 18. 73. 矩阵置零【中等】
**题目描述**  
给定一个 `m x n` 的矩阵，如果一个元素为 `0`，则将其所在行和列的所有元素都设为 `0`。请使用原地算法。
**解题思路**  
用第一行和第一列作为标记数组。先遍历第一行和第一列，判断是否需要最终置零。然后遍历其余矩阵，如果 `matrix[i][j] == 0`，则标记 `matrix[i][0]` 和 `matrix[0][j]` 为 0。最后根据标记数组将对应行和列置零，最后再单独处理第一行和第一列。时间复杂度 O(mn)，空间复杂度 O(1)。

### 19. 54. 螺旋矩阵【中等】
**题目描述**  
给定一个 `m x n` 的矩阵 `matrix`，请按照顺时针螺旋顺序，返回矩阵中的所有元素。
**解题思路**  
模拟路径。维护上下左右四个边界 `top, bottom, left, right`。按顺序从左到右、从上到下、从右到左、从下到上遍历，每完成一个方向就收缩相应的边界。循环直到所有元素被访问。时间复杂度 O(mn)，空间复杂度 O(1)（输出数组不计）。

### 20. 48. 旋转图像【中等】
**题目描述**  
给定一个 `n x n` 的二维矩阵 `matrix` 表示一个图像。请你将图像顺时针旋转 90 度。必须在原地旋转图像，不要使用另一个矩阵来旋转图像。
**解题思路**  
先转置矩阵（沿对角线交换），再翻转每一行。例如：  
1. 转置：`matrix[i][j]` 与 `matrix[j][i]` 交换；  
2. 每行左右翻转：`matrix[i][j]` 与 `matrix[i][n-1-j]` 交换。  
时间复杂度 O(n²)，空间复杂度 O(1)。

### 21. 240. 搜索二维矩阵 II【中等】
**题目描述**  
编写一个高效的算法，在 `m x n` 的矩阵 `matrix` 中搜索一个目标值 `target`。该矩阵具有以下特性：  
- 每行的元素从左到右升序排列。  
- 每列的元素从上到下升序排列。
**解题思路**  
从矩阵右上角开始搜索。若当前值等于 `target`，返回 `true`；若当前值大于 `target`，则该列下方元素更大，可以排除该列（左移）；若当前值小于 `target`，则该行左侧元素更小，可以排除该行（下移）。时间复杂度 O(m+n)，空间复杂度 O(1)。

### 22. 160. 相交链表【简单】
**题目描述**  
给你两个单链表的头节点 `headA` 和 `headB`，请你找出并返回两个单链表相交的起始节点。如果两个链表没有交点，返回 `null`。  
题目保证整个链式结构中不存在环。注意，函数返回结果后，链表必须保持其原始结构。
**解题思路**  
双指针。指针 `pA` 先遍历链表 A，再遍历链表 B；指针 `pB` 先遍历链表 B，再遍历链表 A。两指针走过的总路径长度相同，若有交点，必在交点相遇；若无交点，则同时到达 `null`。时间复杂度 O(m+n)，空间复杂度 O(1)。

### 23. 206. 反转链表【简单】
**题目描述**  
给你单链表的头节点 `head`，请你反转链表，并返回反转后的链表的头节点。
**解题思路**  
迭代法：使用三个指针 `prev`、`curr`、`next`。遍历链表，将 `curr.next` 指向 `prev`，然后依次后移指针。最后 `prev` 就是新的头节点。时间复杂度 O(n)，空间复杂度 O(1)。也可以用递归实现，空间复杂度为 O(n)。

### 24. 234. 回文链表【简单】
**题目描述**  
给你一个单链表的头节点 `head`，请你判断该链表是否为回文链表。如果是，返回 `true`；否则返回 `false`。  
要求时间复杂度 O(n)，空间复杂度 O(1)。
**解题思路**  
快慢指针找到链表中点，反转后半部分链表，然后同时遍历前后两半，逐个比较节点值。若全部相等则是回文链表。最后可选择恢复链表。时间复杂度 O(n)，空间复杂度 O(1)。

### 25. 141. 环形链表【简单】
**题目描述**  
给你一个链表的头节点 `head`，判断链表中是否有环。  
如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。
**解题思路**  
Floyd 判圈算法：快指针每次走两步，慢指针每次走一步。如果有环，两指针一定会相遇；如果无环，快指针会先到达 `null`。时间复杂度 O(n)，空间复杂度 O(1)。

### 26. 142. 环形链表 II【中等】
**题目描述**  
给定一个链表的头节点 `head`，返回链表开始入环的第一个节点。如果链表无环，则返回 `null`。  
不允许修改链表。
**解题思路**  
先用快慢指针判断是否有环并找到相遇点。然后将一个指针移到链表头，两指针以相同速度前进，再次相遇点即为环的入口。原理：设环长为 `c`，相遇点到入口距离为 `a`，起点到入口距离为 `b`，可推导出从起点和相遇点同时出发，会在入口相遇。时间复杂度 O(n)，空间复杂度 O(1)。

### 36. 94. 二叉树的中序遍历【简单】
**题目描述**  
给定一个二叉树的根节点 `root`，返回它的中序遍历结果。
- 前序遍历：先访问「根节点」→ 再访问「左子树」→ 最后访问「右子树」`（根在前）`
- 中序遍历：先访问「左子树」→ 再访问「根节点」→ 最后访问「右子树」`（根在中）`
- 后序遍历：先访问「左子树」→ 再访问「右子树」→ 最后访问「根节点」`（根在后）`

**解题思路**  
递归：利用函数自身调用，天然遵循“左→根→右”的访问顺序——先递归处理左子树，再记录当前节点，最后递归右子树；代码简洁如数学定义，但依赖系统栈，树过深可能爆栈。
迭代：使用栈模拟递归过程。一直向左走并入栈，直到走不通，弹出栈顶访问，然后转向右子树。时间复杂度 O(n)，空间复杂度 O(h)。

### 37. 104. 二叉树的最大深度【简单】
**题目描述**  
给定一个二叉树 `root`，返回其最大深度。二叉树的最大深度是指从根节点到最远叶子节点的最长路径上的节点数。
**解题思路**  
深度搜索优先（推荐）（递归）（DFS）：空节点深度为 0，非空节点的深度 = 左右子树深度的最大值 + 1，`maxDepth(root) = 1 + max(maxDepth(left), maxDepth(right))`。  
广度搜索优先（BFS）：层序遍历，记录层数。时间复杂度 O(n)，空间复杂度 O(h)。

### 58. 39. 组合总和【中等】
**题目描述**

### 84. 72. 编辑距离【困难】
**题目描述**  
给你两个单词 `word1` 和 `word2`，请返回将 `word1` 转换成 `word2` 所使用的最少操作数。你可以对一个单词进行如下三种操作：插入一个字符、删除一个字符、替换一个字符。
**解题思路**  
动态规划：`dp[i][j]` 表示 `word1` 前 `i` 个字符转换到 `word2` 前 `j` 个字符的最少操作数。若字符相等，`dp[i][j] = dp[i-1][j-1]`；否则 `dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])`，分别对应删除、插入、替换。时间复杂度 O(mn)。

### 85. 121. 买卖股票的最佳时机【简单】
**题目描述**

**解题思路**

### 86. 55. 跳跃游戏【中等】
**题目描述**  
给定一个非负整数数组 `nums`，你最初位于数组的第一个下标。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标。
**解题思路**  
维护一个变量 `maxReach` 表示当前能到达的最远下标。遍历数组，如果当前位置 `i` 超过 `maxReach`，则返回 `false`；否则更新 `maxReach = max(maxReach, i + nums[i])`。如果 `maxReach` 大于等于最后一个下标，返回 `true`。时间复杂度 O(n)。

### 87. 45. 跳跃游戏 II【中等】
**题目描述**  
给定一个非负整数数组 `nums`，你最初位于数组的第一个下标。数组中的每个元素代表你在该位置可以跳跃的最大长度。你的目标是使用最少的跳跃次数到达数组的最后一个下标。假设总能到达数组的最后一个下标。
**解题思路**  
贪心。维护当前步能覆盖的边界 `end` 和下一步能覆盖的最大范围 `maxPos`。遍历时更新 `maxPos`，当 `i == end` 时，说明这一步走完了，步数 `+1`，更新 `end = maxPos`。时间复杂度 O(n)。

### 88. 763. 划分字母区间【中等】
**题目描述**  
字符串 `S` 由小写字母组成。我们要把这个字符串划分为尽可能多的片段，同一字母最多出现在一个片段中。返回一个表示每个字符串片段长度的列表。
**解题思路**

### 89. 155. 最小栈【中等】
**题目描述**  
设计一个支持 `push`、`pop`、`top` 操作，并能在常数时间内检索到最小元素的栈。  
- `push(x)` 将元素 x 推入栈中。  
- `pop()` 删除栈顶的元素。

### 93. 84. 柱状图中最大的矩形【困难】

### 96. 295. 数据流的中位数【困难】
**题目描述**  
中位数是有序整数列表中的中间值。如果列表的大小是偶数，则没有中间值，中位数是两个中间值的平均值。  
设计支持以下两种操作的数据结构：  
- `void addNum(int num)` 从数据流中添加一个整数到数据结构中；  
- `double findMedian()` 返回目前所有元素的中位数。
**解题思路**  
双堆法。一个大顶堆存较小的一半，一个小顶堆存较大的一半。通过调整两堆的大小关系，使得中位数可以通过堆顶元素直接计算。`addNum` 时根据堆顶大小关系决定插入哪个堆，并平衡大小。`findMedian` 时根据两堆大小决定返回堆顶或平均值。时间复杂度 O(log n)。

### 97. 136. 只出现一次的数字【简单】
**题目描述**  
给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。
**解题思路**  
异或运算。利用 `a ^ a = 0`，`a ^ 0 = a`。将所有数字异或，成对数字会抵消为 0，剩下的就是只出现一次的数字。时间复杂度 O(n)，空间复杂度 O(1)。

### 98. 169. 多数元素【简单】
**题目描述**  
给定一个大小为 `n` 的数组 `nums`，返回其中的多数元素。多数元素是指在数组中出现次数大于 `⌊ n/2 ⌋` 的元素。你可以假设数组是非空的，并且给定的数组总是存在多数元素。
**解题思路**  
Boyer-Moore 投票算法。维护候选人和计数器。遍历数组，如果计数为 0，更换候选人；如果当前元素等于候选人，计数 `+1`；否则计数 `-1`。最后剩下的候选人即为多数元素。时间复杂度 O(n)，空间复杂度 O(1)。

### 99. 75. 颜色分类【中等】
**题目描述**  
给定一个包含红色、白色和蓝色，共 `n` 个元素的数组 `nums`，原地对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。这里使用整数 `0`、`1` 和 `2` 分别表示红色、白色和蓝色。
**解题思路**  
三指针（荷兰国旗问题）。使用三个指针 `p0`、`p2`、`curr`。遍历数组，遇到 `0` 与 `p0` 交换并移动指针，遇到 `2` 与 `p2` 交换（此时 `curr` 不动），遇到 `1` 直接移动。时间复杂度 O(n)，空间复杂度 O(1)。

### 100. 31. 下一个排列【中等】
**题目描述**  
实现获取下一个排列的函数，算法需要将给定数字序列重新排列成字典序中下一个更大的排列。如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。必须原地修改，只允许使用常数空间。
**解题思路**  
从后向前找第一对升序对 `nums[i] < nums[i+1]`，再从后向前找第一个大于 `nums[i]` 的数 `nums[j]`，交换两者，最后反转 `i+1` 到末尾的部分。时间复杂度 O(n)，空间复杂度 O(1)。
