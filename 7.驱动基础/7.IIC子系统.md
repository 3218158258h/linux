## 框架概述
### I²C子系统整体架构
```
应用层 (Applications)
    ↓ (ioctl, sysfs, etc.)
内核空间 (Kernel Space)
├── I²C Core (核心层)
│   ├── 总线管理 (Bus Management)
│   ├── 适配器/设备/驱动注册 (i2c_add_adapter/i2c_del_adapter、i2c_register_driver/i2c_unregister_driver)
│   └── 数据传输 (Transfer Handling)
├── I²C Adapter (适配器层 - BSP)
│   ├── 硬件控制器驱动 (Hardware Controller Driver)
│   └── 通信协议实现 (Communication Protocol Implementation)
└── I²C Client Drivers (客户端驱动层 - 驱动开发者重点)
    ├── 具体设备驱动 (e.g., sensors, EEPROMs, RTCs)
    └── 与I2C Core交互
``` 

**i2c_adapter（适配器 主机）**
- 代表 I2C 主机控制器，负责真正和从设备物理通信、发时序、传数据。

**i2c_client(从机)**
- 代表 挂在总线上的从设备（传感器、EEPROM、触摸屏等），是从设备在内核里的 “化身”，内核自动识别设备树节点并创建该结构体，绑定到对应的适配器上。

**i2c_driver（驱动层）**
- 对应具体外设的驱动，负责业务逻辑：初始化、读数据、写控制、中断处理等。

**i2c_bus_type（总线）**
- 是一条虚拟总线 负责匹配 i2c_driver 和 i2c_client，匹配成功就调用 probe。

## 核心层 (I²C Core)

I²C核心层是整个子系统的核心，它提供了一套标准的API，使得上层的驱动程序可以不关心底层硬件的具体实现细节。核心层的主要职责包括：

- **总线管理**：注册和注销I²C总线。
- **设备发现**：通过设备树或板级文件信息，将I²C设备添加到内核中。
- **驱动匹配**：根据设备地址和名称等信息，将设备与对应的驱动进行绑定。
- **消息传输**：提供`i2c_transfer()`等函数，用于在适配器上发送和接收消息。

### 关键结构体
- `struct i2c_adapter`: 描述一个I²C适配器。
- `struct i2c_client`: 描述一个连接到总线上的I²C设备。
- `struct i2c_driver`: 描述一个I²C设备驱动。

### 核心API
1、i2c_adapter 注册/注销函数
`int i2c_add_adapter(struct i2c_adapter *adapter)`
`void i2c_del_adapter(struct i2c_adapter * adap)`
2、i2c_driver 注册/注销函数
`int i2c_register_driver(struct module *owner, struct i2c_driver *driver)`
`int i2c_add_driver (struct i2c_driver *driver)`
`void i2c_del_driver(struct i2c_driver *driver)`
3、i2c_transfer 数据传输函数
`int i2c_transfer(struct i2c_adapter *adap, struct i2c_msg *msgs, int num)`: 执行数据传输。

### `struct i2c_bus_type`结构体
- 定义了I²C总线类型，包含匹配函数和总线名称等信息,用于配对i2c_driver和i2c_client，配对成功后调用probe函数。
```c
struct bus_type i2c_bus_type = {
	.name		= "i2c",
	.match		= i2c_device_match,
	.probe		= i2c_device_probe,    // 总线层 probe
	.remove		= i2c_device_remove,
	.shutdown	= i2c_device_shutdown,
	.pm		= &i2c_device_pm_ops,
};
EXPORT_SYMBOL_GPL(i2c_bus_type);
```

## 适配器驱动层 (I²C Adapter Driver)

- I2C适配器代表了板子上的主机（I²C主机控制器），负责和从机设备通信，通过`struct i2c_adapter`结构体描述一个I²C适配器。
- 结构体内部有`struct i2c_algorithm`结构体，用于描述I²C协议传输算法，包括`master_xfer`和`smbus_xfer`函数。`struct i2c_msg`结构体用于描述消息

- I²C适配器层主要由`BSP开发者`负责实现，针对特定SoC的I²C控制器编写驱动程序，并将其注册到I²C核心层。
- 适配器层的实现需要处理具体硬件的寄存器操作、时序控制等细节。驱动开发者不用关心适配器层的实现细节，只需要使用核心层提供的API进行数据传输

### 结构体详解
#### `struct i2c_client`结构体详解
- 代表一个从机设备，包含设备地址、适配器、驱动等信息。内核自动识别设备树节点并创建该结构体，绑定到对应的适配器上。并在配对驱动后传入到 `probe`函数中。
- 不归驱动层管理，由内核自动创建和销毁。属于核心层，驱动只是使用者，不是创建者
```c
#include <linux/i2c.h>
struct i2c_client {
	unsigned short		flags;		/* 标志位（10bit 地址等） */
	unsigned short		addr;		/* I2C 设备地址（7bit） */
	struct i2c_adapter	*adapter;	/* 所属 I2C 适配器/总线 */
	struct i2c_driver	*driver;	/* 绑定的驱动 */
	struct device		dev;		/* 设备模型基类 */
	char		name[I2C_NAME_SIZE];	/* 设备名 */
	struct mutex		lock;		/* 传输互斥锁 */
    // 其他字段...
};
```


#### `struct i2c_adapter`结构体
- 代表一个I²C适配器，包含适配器的基本信息和指向传输算法的指针。
```c
#include <linux/i2c.h>
struct i2c_adapter {
    struct module *owner;
    const struct i2c_algorithm *algo;   /* 核心！I2C 传输算法 */
    void *algo_data;                     /* 算法私有数据 */
    struct device dev;                   /* 内嵌设备结构体 */
    int nr;                              /* 总线号nr=0 i2c-0,nr=1 i2c-1... */
    char name[48];                       /* 适配器名字 */
    // 其他字段...
};
```
##### `struct i2c_algorithm *algo`结构体
- I2C传输算法，定义了I2C传输的函数。
```c
struct i2c_algorithm {
    /* I2C 传输函数：控制器真正收发数据 */
    //负责产生时钟、发送地址、读写数据
    //adap：适配器 msgs：I2C 消息数组 num：消息数量
    int (*master_xfer)(struct i2c_adapter *adap, struct i2c_msg *msgs, int num);
    
    //SMBUS 总线的传输函数。
    int (*smbus_xfer)(struct i2c_adapter *adap, u16 addr,
                      unsigned short flags, char read_write,
                      u8 command, int size, union i2c_smbus_data *data);

    /* 返回适配器支持哪些功能（I2C_FUNC_I2C等）*/
    u32 (*functionality)(struct i2c_adapter *);
};
```


### 相关函数
- `int i2c_add_adapter(struct i2c_adapter * adap)`: 将适配器`struct i2c_adapter`适配器注册到内核,成功返回0,失败返回负值
- `void i2c_del_adapter(struct i2c_adapter * adap)`: 从内核中删除适配器



## 设备驱动层 (I²C Client Driver)

这是驱动开发者工作的主要区域。需要为具体的I²C设备（如传感器、EEPROM、RTC等）编写驱动程序。

### 实现流程

1. **定义`struct i2c_driver`**：
   - `.probe()`: 设备驱动匹配成功后调用，初始化设备。
   - `.remove()`: 设备移除时调用，清理资源。
   - `.id_table`: 匹配表，用于旧版匹配设备号
   - `.driver`: 内嵌设备驱动结构体，设置驱动名称、所属模块等信息。
   - `.of_match_table`: 设备树匹配表（如果使用设备树）。

2. **实现`.probe()`函数**：
   - 获取`struct i2c_client`指针。
   - 读取设备寄存器以确认设备存在。
   - 初始化设备配置。
   - 注册字符设备、输入设备等（如果需要）。

3. **实现读写操作**：
   - 使用`i2c_smbus_read_byte_data()`, `i2c_smbus_write_byte_data()`等函数与设备通信。
   - 或者使用更底层的`i2c_transfer()`函数构建复杂的传输序列。

4. **注册驱动**：
   - 在模块加载函数中调用`i2c_register_driver()`。

### `struct i2c_driver`结构体详解
```c
#include <linux/i2c.h>
struct i2c_driver {
    /* 必须：设备驱动的通用基类（所有总线驱动都继承它） */
    struct device_driver driver;
    /* 必须：匹配成功后立即执行 */
    int (*probe)(struct i2c_client *client, const struct i2c_device_id *id);
    /* 必须：设备移除时的清理回调函数 */
    void (*remove)(struct i2c_client *client);

    /* 可选：设备树匹配表（用于设备树匹配设备） */
    const struct of_device_id *of_match_table;
    /* 可选：传统 ID 匹配表（非设备树场景使用） */
    const struct i2c_device_id *id_table;
    /* 可选：关机时的回调 */
    void (*shutdown)(struct i2c_client *client);
    /* 可选：电源管理操作（休眠/唤醒） */
    const struct dev_pm_ops *pm;
    /* 私有数据，内核内部使用，驱动不用管 */
    struct list_head clients;
};
```


### `int i2c_transfer`函数
- 用于在驱动层上发送和接收消息，最终会调用适配器层的`master_xfer`函数进行实际的数据传输
- i2c_transfer 是 Linux I2C 子系统最底层、最通用的收发接口
- 所有 I2C 读写函数（i2c_smbus_read_byte_data、i2c_master_send等）最终都是调用它实现的。
```c
#include <linux/i2c.h>
/*adap：所使用的 I2C 适配器，i2c_client 会保存其对应的 i2c_adapter。
msgs：I2C 要发送的一个或多个消息。
num：消息数量，也就是 msgs 的数量*/
int i2c_transfer(struct i2c_adapter *adap, struct i2c_msg *msgs, int num);
```
**成功**：返回传输成功的 message 数量（=num）

**失败**：返回负数（错误码）

### `i2c_msg`结构体
- 描述一个I²C消息，包括从设备地址、读/写标志、数据长度和数据缓冲区。
```c
struct i2c_msg {
    u16 addr;   // 从设备地址（7位或10位）
    u16 flags;  // 读/写标志：I2C_M_RD 表示读
    u16 len;    // 数据长度
    u8 *buf;    // 数据缓冲区
};
```

### 示例代码骨架
```c
static int device_probe(struct i2c_client *client, const struct i2c_device_id *id)
{
    // 1. 检查设备是否存在
    if (!i2c_check_functionality(client->adapter, I2C_FUNC_SMBUS_BYTE_DATA))
        return -ENODEV;
    // 2. 初始化设备
    i2c_smbus_write_byte_data(client, REG_CONFIG, CONFIG_VALUE);
    // 3. 注册设备 (例如，input_dev, miscdevice, etc.)
    // ...
    dev_info(&client->dev, "Device probed successfully\n");
    return 0;
}
static int device_remove(struct i2c_client *client)
{
    // 清理资源
    return 0;
}
static const struct i2c_device_id device_id[] = {
    { "my_device_name", 0 },
    { }
};
static struct i2c_driver my_i2c_driver = {
    .driver = {
        .name = "my_device_name",
        .owner = THIS_MODULE,
        .bus = &i2c_bus_type,
    },
    .probe = device_probe,
    .remove = device_remove,
    .id_table = device_id,
};
module_i2c_driver(my_i2c_driver); // 这个宏自动处理 module_init/module_exit
```

### 调用`i2c_transfer`读写代码示例
```c
/* 设备结构体 */
struct xxx_dev {
    ......
    void *private_data; /* 私有数据，一般会设置为 i2c_client */
};

/*
 * @description: 读取 I2C 设备多个寄存器数据
 * @param – dev : 自定义的I2C设备结构体
 * @param – reg : 要读取的寄存器首地址
 * @param – val : 读取到的数据存放缓冲区
 * @param – len : 要读取的数据长度
 * @return : 操作结果 0成功，负数失败
 */
static int xxx_read_regs(struct xxx_dev *dev, u8 reg, void *val, int len)
{
    int ret;
    struct i2c_msg msg[2];//先写入寄存器地址，再读取数据，所以需要两条消息
    struct i2c_client *client = (struct i2c_client *)dev->private_data;

    /* msg[0]，第一条写消息，发送要读取的寄存器首地址 */
    msg[0].addr  = client->addr;  /* I2C 器件地址 */
    msg[0].flags = 0;             /* 标记为发送数据 */
    msg[0].buf   = &reg;          /* 读取的首地址 */
    msg[0].len   = 1;             /* reg 长度 */

    /* msg[1]，第二条读消息，读取寄存器数据 */
    msg[1].addr  = client->addr;  /* I2C 器件地址 */
    msg[1].flags = I2C_M_RD;      /* 标记为读取数据 */
    msg[1].buf   = val;           /* 接收数据的缓冲区 */
    msg[1].len   = len;           /* 要读取的数据长度 */

    ret = i2c_transfer(client->adapter, msg, 2);
    if(ret == 2) {
        ret = 0;
    } else {
        ret = -EREMOTEIO;
    }
    return ret;
}


/*
 * @description : 向 I2C 设备多个寄存器写入数据
 * @param – dev : 要写入的设备结构体
 * @param – reg : 要写入的寄存器首地址
 * @param – buf : 要写入的数据缓冲区
 * @param – len : 要写入的数据长度
 * @return : 操作结果
 */
static s32 xxx_write_regs(struct xxx_dev *dev, u8 reg, u8 *buf, u8 len)
{
    u8 b[256];
    struct i2c_msg msg;//同时写入寄存器地址和数据，所以只需要一条消息
    struct i2c_client *client = (struct i2c_client *)dev->private_data;

    b[0] = reg; /* 寄存器首地址 */
    memcpy(&b[1],buf,len); /* 将要发送的数据拷贝到数组 b 里面 */

    msg.addr = client->addr; /* I2C 器件地址 */
    msg.flags = 0; /* 标记为写数据 */

    msg.buf = b; /* 要发送的数据缓冲区 */
    msg.len = len + 1; /* 要发送的数据长度 */

    return i2c_transfer(client->adapter, &msg, 1);
}
```

### `i2c_master_send` 函数
- 用于发送 I2C 消息,底层也是调用 `i2c_transfer` 实现的
```c
int i2c_master_send(const struct i2c_client *client,
                    const char *buf,
                    int count);
```
- client：I2C 设备结构体指针。
- buf：要发送的数据缓冲区
- count：要发送的字节数
- 返回值：成功返回发送的字节数，失败返回负数（错误码）


### `i2c_master_recv` 函数
- 用于接收 I2C 消息,底层也是调用 `i2c_transfer` 实现的
```c
int i2c_master_recv(const struct i2c_client *client,
                    char *buf,
                    int count);
```
- client：I2C 设备结构体指针。
- buf：要接收的数据缓冲区
- count：要接收的字节数
- 返回值：成功返回接收的字节数，失败返回负数（错误码）
