I²C（Inter-Integrated Circuit）是一种同步、多主机、多从机的串行通信总线，广泛应用于嵌入式系统中连接低速外围设备。

## 框架概述

### I²C子系统整体架构
```
应用层 (Applications)
    ↓ (ioctl, sysfs, etc.)
内核空间 (Kernel Space)
├── I²C Core (核心层)
│   ├── 总线管理 (Bus Management)
│   ├── 适配器/设备/驱动注册 (i2c_add_adapter/i2c_del_adapter、i2c_register_driver/i2c_unregister_driver)
│   └── 数据传输 (Transfer Handling)
├── I²C Adapter (适配器层 - BSP)
│   ├── 硬件控制器驱动 (Hardware Controller Driver)
│   └── 物理接口实现 (Physical Interface Implementation)
└── I²C Client Drivers (客户端驱动层 - 驱动开发者重点)
    ├── 具体设备驱动 (e.g., sensors, EEPROMs, RTCs)
    └── 与I2C Core交互
```


**i2c_adapter（适配器 主机）**
- 代表 I2C 主机控制器，负责真正和从设备物理通信、发时序、传数据。

**i2c_client(从机)**
- 代表 挂在总线上的从设备（传感器、EEPROM、触摸屏等），是从设备在内核里的 “化身”。

**i2c_driver（驱动层）**
- 对应具体外设的驱动，负责业务逻辑：初始化、读数据、写控制、中断处理等。

**i2c_bus_type（总线）**
- 负责匹配 i2c_driver 和 i2c_client，匹配成功就调用 probe。
- 是SCL (时钟) 和 SDA (数据) 两条信号线，是真实的总线。

## 核心层 (I²C Core)

I²C核心层是整个子系统的核心，它提供了一套标准的API，使得上层的驱动程序可以不关心底层硬件的具体实现细节。核心层的主要职责包括：

- **总线管理**：注册和注销I²C总线。
- **设备发现**：通过设备树或板级文件信息，将I²C设备添加到系统中。
- **驱动匹配**：根据设备地址和名称等信息，将设备与对应的驱动进行绑定。
- **消息传输**：提供`i2c_transfer()`等函数，用于在适配器上发送和接收消息。

### 关键结构体
- `struct i2c_adapter`: 描述一个I²C适配器。
- `struct i2c_client`: 描述一个连接到总线上的I²C设备。
- `struct i2c_driver`: 描述一个I²C设备驱动。

### 核心API
- `int i2c_register_driver(struct module *owner, struct i2c_driver *driver)`: 注册一个I²C驱动。
- `int i2c_add_driver(struct i2c_driver *driver)`: 添加I²C驱动。
- `void i2c_del_driver(struct i2c_driver *driver)`: 删除I²C驱动。
- `int i2c_transfer(struct i2c_adapter *adap, struct i2c_msg *msgs, int num)`: 执行数据传输。

## 适配器层 (I²C Adapter Driver)

- I2C适配器代表了板子上的主机（I²C主机控制器），负责和从机设备通信，通过`struct i2c_adapter`结构体描述一个I²C适配器。
- 结构体内部有`struct i2c_algorithm`结构体，用于描述I²C协议传输算法，包括`master_xfer`和`smbus_xfer`函数。`struct i2c_msg`结构体用于描述消息

- I²C适配器层主要由BSP开发者负责实现，针对特定SoC的I²C控制器编写驱动程序，并将其注册到I²C核心层。
- 适配器层的实现需要处理具体硬件的寄存器操作、时序控制等细节。驱动开发者不用关心适配器层的实现细节，只需要使用核心层提供的API进行数据传输

### 结构体详解
#### `struct i2c_adapter`结构体
- 代表一个I²C适配器，包含适配器的基本信息和指向传输算法的指针。
```c
#include <linux/i2c.h>
struct i2c_adapter {
    struct module *owner;
    const struct i2c_algorithm *algo;   /* 核心！I2C 传输算法 */
    void *algo_data;                     /* 算法私有数据 */
    struct device dev;                   /* 内嵌设备结构体 */
    int nr;                              /* 总线号nr=0 i2c-0,nr=1 i2c-1... */
    char name[48];                       /* 适配器名字 */
    // 其他字段...
};
```
##### `struct i2c_algorithm *algo`结构体
- I2C传输算法，定义了I2C传输的函数。
```c
struct i2c_algorithm {
    /* I2C 传输函数：控制器真正收发数据 */
    //负责产生时钟、发送地址、读写数据
    //adap：适配器 msgs：I2C 消息数组 num：消息数量
    int (*master_xfer)(struct i2c_adapter *adap, struct i2c_msg *msgs, int num);
    
    //SMBUS 总线的传输函数。
    int (*smbus_xfer)(struct i2c_adapter *adap, u16 addr,
                      unsigned short flags, char read_write,
                      u8 command, int size, union i2c_smbus_data *data);

    /* 返回适配器支持哪些功能（I2C_FUNC_I2C等）*/
    u32 (*functionality)(struct i2c_adapter *);
};
```

###### `i2c_msg`结构体
- 描述一个I²C消息，包括从设备地址、读/写标志、数据长度和数据缓冲区。
```c
struct i2c_msg {
    u16 addr;   // 从设备地址（7位或10位）
    u16 flags;  // 读/写标志：I2C_M_RD 表示读
    u16 len;    // 数据长度
    u8 *buf;    // 数据缓冲区
};
```
### 相关函数
- `int i2c_add_adapter(struct i2c_adapter * adap)`: 将适配器`struct i2c_adapter`适配器注册到内核,成功返回0,失败返回负值
- `void i2c_del_adapter(struct i2c_adapter * adap)`: 从内核中删除适配器

## 客户端驱动层 (I²C Client Driver)

这是驱动开发者工作的主要区域。需要为具体的I²C设备（如传感器、EEPROM、RTC等）编写驱动程序。

### 编写流程

1. **定义`struct i2c_driver`**：
   - `.probe()`: 设备探测成功后调用，初始化设备。
   - `.remove()`: 设备移除时调用，清理资源。
   - `.id_table`: 匹配表，列出驱动支持的设备名称和ID。

2. **实现`.probe()`函数**：
   - 获取`struct i2c_client`指针。
   - 读取设备寄存器以确认设备存在。
   - 初始化设备配置。
   - 注册字符设备、输入设备等（如果需要）。
   - 创建sysfs属性文件（如果需要）。

3. **实现读写操作**：
   - 使用`i2c_smbus_read_byte_data()`, `i2c_smbus_write_byte_data()`等函数与设备通信。
   - 或者使用更底层的`i2c_transfer()`函数构建复杂的传输序列。

4. **注册驱动**：
   - 在模块加载函数中调用`i2c_register_driver()`。

### 示例代码骨架
```c
static int device_probe(struct i2c_client *client, const struct i2c_device_id *id)
{
    // 1. 检查设备是否存在
    if (!i2c_check_functionality(client->adapter, I2C_FUNC_SMBUS_BYTE_DATA))
        return -ENODEV;

    // 2. 初始化设备
    i2c_smbus_write_byte_data(client, REG_CONFIG, CONFIG_VALUE);

    // 3. 注册设备 (例如，input_dev, miscdevice, etc.)
    // ...

    dev_info(&client->dev, "Device probed successfully\n");
    return 0;
}

static int device_remove(struct i2c_client *client)
{
    // 清理资源
    return 0;
}

static const struct i2c_device_id device_id[] = {
    { "my_device_name", 0 },
    { }
};

static struct i2c_driver my_i2c_driver = {
    .driver = {
        .name = "my_device_name",
    },
    .probe = device_probe,
    .remove = device_remove,
    .id_table = device_id,
};

module_i2c_driver(my_i2c_driver); // 这个宏自动处理 module_init/module_exit
```

### 关键点
- **设备树集成**：在设备树中为你的设备添加节点，并将其链接到正确的I²C总线（`i2c@...`节点下）。这将触发`.probe()`函数被调用。
- **地址分配**：确保你知道设备的I²C地址（通常在数据手册中），并正确设置`i2c_client`的`addr`字段（通常由设备树自动填充）。
- **错误处理**：正确处理硬件访问失败的情况。
- **电源管理**：如果设备支持，应实现挂起(suspend)/恢复(resume)回调。
