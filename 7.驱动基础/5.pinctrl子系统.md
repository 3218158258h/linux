## 一、基础概念

### 1. 什么是 pinctrl？
- **pinctrl** 是 Linux 内核中用于管理 SoC 引脚复用和引脚配置的子系统。
- 它允许驱动在运行时动态地配置引脚的功能（如 GPIO、I2C、UART 等）以及电气特性（如上拉/下拉、驱动强度、输入/输出模式等）。
- 获取pin的配置
- 根据pin的配置复用引脚
- 配置pin的电气属性
### 2. 为什么需要 pinctrl？
- 现代 SoC 引脚数量有限，但外设功能众多，一个物理引脚可能支持多种功能（复用）。
- 不同外设对引脚电气属性要求不同（如 I2C 需要开漏、上拉电阻等）。
- pinctrl 提供统一接口，避免每个驱动重复实现引脚配置逻辑。

### 3. 相关术语
- **Pin**：物理引脚。
- **Function**：引脚可复用的功能（如 “uart0_tx”、“i2c1_scl”）。
- **Group**：一组引脚，通常对应某个功能所需的所有引脚（如 “uart0_grp”）。
- **State**：设备在不同工作状态下所需的引脚配置（如 “default”、“sleep”）。
- **Pin Configuration**：引脚的电气属性设置（pull-up/down、slew rate、drive strength 等）。

## 二、架构与核心组件

### 1. pinctrl 子系统架构
- **Core Layer**（核心层）：`drivers/pinctrl/core.c`
  - 提供通用框架、注册/注销机制、状态机管理。
- **Pin Controller Driver**（控制器驱动）：
  - 每个 SoC 厂商需实现自己的 pinctrl 驱动（如 `pinctrl-rockchip.c`, `pinctrl-am335x.c`）。
  - 实现 `struct pinctrl_desc` 和操作函数（`pinmux_ops`, `pinconf_ops`）。
- **Client Driver**（客户端驱动）：
  - 使用 pinctrl 接口的设备驱动（如 UART、SPI 驱动）。
  - 通过 `devm_pinctrl_get()` 获取 pinctrl 句柄，并切换状态。

### 2. 关键数据结构
- `struct pinctrl_desc`：描述整个 pin controller 的能力。
- `struct pinctrl_dev`：代表一个已注册的 pin controller 设备。
- `struct pinctrl_state`：表示一种引脚配置状态（如 default/sleep）。
- `struct pinctrl`：客户端使用的句柄，关联设备和状态。
- `struct pinctrl_setting`：具体的一组引脚配置（function + config）。

### 3. 核心 API（供客户端驱动使用）
```c
pinctrl = devm_pinctrl_get(&pdev->dev);
state = pinctrl_lookup_state(pinctrl, "default");
pinctrl_select_state(pinctrl, state);
```
- `devm_pinctrl_get()` / `pinctrl_put()`
- `pinctrl_lookup_state()`
- `pinctrl_select_state()`
- `pinctrl_pm_select_default_state()`（电源管理集成）

## 三、设备树（Device Tree）集成

### 1. pinctrl 节点定义
在 `.dtsi` 或 `.dts` 中定义引脚配置：

```dts
&uart1 {
    pinctrl-names = "default", "sleep";
    pinctrl-0 = <&uart1_pins>;
    pinctrl-1 = <&uart1_sleep_pins>;
    status = "okay";
};

&pinctrl {
    uart1_pins: uart1-pins {
        pins = "GPIO_40", "GPIO_41";
        function = "uart1";
        bias-pull-up;
        drive-strength = <8>;
    };

    uart1_sleep_pins: uart1-sleep-pins {
        pins = "GPIO_40", "GPIO_41";
        function = "gpio";
        bias-disable;
    };
};
```

### 2. 常见属性
- `pins`：引脚名称列表。
- `function`：复用功能。
- `bias-pull-up` / `bias-pull-down` / `bias-disable`
- `drive-strength`：驱动电流（mA）
- `input-enable` / `output-high` / `output-low`
- `slew-rate`：压摆率

> 注意：具体属性由 SoC 厂商定义，需参考其 pinctrl 驱动实现。

## 四、pinctrl 驱动开发（SoC 厂商视角）

### 1. 实现步骤
1. 定义引脚描述（`struct pinctrl_pin_desc`）
2. 实现 `pinmux_ops`（复用控制）：
   - `get_functions_count()`
   - `get_function_name()`
   - `get_function_groups()`
   - `set_mux()`
3. 实现 `pinconf_ops`（配置控制）：
   - `pin_config_get()`
   - `pin_config_set()`
   - 支持标准配置参数（`PIN_CONFIG_BIAS_PULL_UP` 等）
4. 填充 `struct pinctrl_desc`
5. 调用 `devm_pinctrl_register()` 注册控制器

### 2. 示例代码骨架
```c
static const struct pinctrl_ops my_pctrl_ops = { ... };
static const struct pinmux_ops my_pinmux_ops = { ... };
static const struct pinconf_ops my_pinconf_ops = { ... };

static struct pinctrl_desc my_pinctrl_desc = {
    .name = "my-pinctrl",
    .pins = my_pins,
    .npins = ARRAY_SIZE(my_pins),
    .pctlops = &my_pctrl_ops,
    .pmxops = &my_pinmux_ops,
    .confops = &my_pinconf_ops,
    .owner = THIS_MODULE,
};

static int my_pinctrl_probe(struct platform_device *pdev)
{
    return devm_pinctrl_register_and_init(&pdev->dev, &my_pinctrl_desc, NULL, &pcdev);
}
```

## 五、调试与工具

### 1. sysfs 接口（若启用）
- `/sys/kernel/debug/pinctrl/` 下有各控制器信息
  - `pinmux`
  - `pins`
  - `groups`
  - `functions`

### 2. 常用调试命令
```bash
cat /sys/kernel/debug/pinctrl/*/pinmux
dmesg | grep pinctrl
```

### 3. 常见问题
- 引脚冲突（多个设备尝试配置同一引脚）
- 设备树未正确引用 pinctrl 状态
- pinctrl 驱动未正确实现 `set_mux`
- 电源状态切换时未恢复引脚配置

## 六、进阶主题

### 1. 与 GPIO 子系统的关系
- GPIO 是 pinctrl 的一种“function”。
- 当引脚配置为 GPIO 时，由 gpiochip 控制方向/电平；pinctrl 负责将其复用为 GPIO 功能并配置电气属性。

### 2. 电源管理集成
- pinctrl 支持 suspend/resume 时自动切换到 “sleep” 状态，resume 时切回 “default”。

### 3. 多状态支持
- 一个设备可定义多个状态（如 init, active, idle, sleep），由驱动按需切换。

### 4. generic pinconfig
- 内核提供通用解析器（`pinconf-generic.c`），支持从设备树自动解析标准属性。
