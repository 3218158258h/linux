## pinctrl子系统框架

### 1. pinctrl的作用和意义
- **统一管理**：SoC引脚复用和电气配置的统一接口
- **避免重复**：防止每个外设驱动都自己实现引脚配置
- **运行时切换**：支持引脚功能在运行时动态切换

### 2. pinctrl核心概念

#### pinctrl的三个主要功能
- **引脚复用（Pin Multiplexing）**：一个引脚可配置为多种功能（GPIO/I2C/SPI等）
- **电气特性配置**：上下拉电阻、驱动强度、 slew rate等
- **状态管理**：不同工作状态下的引脚配置（默认/休眠/备份等）

#### 三个主要参与方
- **Client驱动**：使用pinctrl的外设驱动（如I2C、SPI、UART驱动）
- **Core层**：内核提供的pinctrl框架
- **Controller驱动**：SoC厂商提供的具体pinctrl实现

### 3. pinctrl使用框架

#### A. 设备树配置（定义引脚组）
```c
pinctrl {
    uart1 {
        uart1_default: uart1-default {
            rockchip,pins = <1 RK_PA2 2 &pcfg_pull_none>,
                          <1 RK_PA3 2 &pcfg_pull_none>;
        };
        
        uart1_sleep: uart1-sleep {
            rockchip,pins = <1 RK_PA2 0 &pcfg_pull_down>,
                          <1 RK_PA3 0 &pcfg_pull_down>;
        };
    };
};

&uart1 {
    pinctrl-names = "default", "sleep";
    pinctrl-0 = <&uart1_default>;  // 默认状态
    pinctrl-1 = <&uart1_sleep>;    // 休眠状态
};
```

#### B. 外设驱动使用pinctrl
```c
static int my_uart_probe(struct platform_device *pdev)
{
    struct my_uart_dev *dev;
    struct pinctrl *pinctrl;
    struct pinctrl_state *state;
    
    // 1. 获取pinctrl句柄
    pinctrl = devm_pinctrl_get(&pdev->dev);
    if (IS_ERR(pinctrl)) {
        return PTR_ERR(pinctrl);
    }
    
    // 2. 获取指定状态的引脚配置
    state = pinctrl_lookup_state(pinctrl, PINCTRL_STATE_DEFAULT);
    if (IS_ERR(state)) {
        return PTR_ERR(state);
    }
    
    // 3. 切换到该状态（配置引脚）
    return pinctrl_select_state(pinctrl, state);
}
```

#### C. 核心API汇总
```c
// 获取pinctrl句柄
struct pinctrl *devm_pinctrl_get(struct device *dev);

// 查找状态
struct pinctrl_state *pinctrl_lookup_state(struct pinctrl *p, const char *name);

// 切换状态
int pinctrl_select_state(struct pinctrl *p, struct pinctrl_state *s);

// 电源管理集成
int pinctrl_pm_select_default_state(struct device *dev);
int pinctrl_pm_select_sleep_state(struct device *dev);
```

### 4. pinctrl子系统架构

#### 数据结构关系
```
struct pinctrl_desc (控制器描述) 
    ├── pins[] (引脚描述数组)
    ├── pctlops (基础操作集)
    ├── pmxops (复用操作集) 
    └── confops (配置操作集)

struct pinctrl_dev (注册后的设备实例)

struct pinctrl (客户端句柄)
    └── states[] (状态数组)
        └── struct pinctrl_state (具体状态)
```

### 5. Groups和Functions概念

#### Groups（引脚组）
- **定义**：物理上相关的引脚集合
- **用途**：便于批量配置相关引脚
- **示例**：I2C的SDA和SCL引脚组成一个组

#### Functions（功能）
- **定义**：引脚可实现的复用功能
- **用途**：标识引脚可配置为何种外设功能
- **示例**：引脚可配置为GPIO功能或I2C功能

### 6. 外设驱动中pinctrl使用模板

```c
static const struct of_device_id my_driver_of_match[] = {
    { .compatible = "vendor,my-device", },
    { }
};

static struct platform_driver my_driver = {
    .probe = my_driver_probe,
    .remove = my_driver_remove,
    .driver = {
        .name = "my-driver",
        .of_match_table = my_driver_of_match,
    },
};

static int my_driver_probe(struct platform_device *pdev)
{
    struct device *dev = &pdev->dev;
    struct pinctrl *pinctrl;
    
    // pinctrl处理 - 通常在其他初始化之前
    pinctrl = devm_pinctrl_get(dev);
    if (!IS_ERR(pinctrl)) {
        struct pinctrl_state *state;
        
        // 获取默认状态
        state = pinctrl_lookup_state(pinctrl, "default");
        if (!IS_ERR(state))
            pinctrl_select_state(pinctrl, state);
    }
    
    // 其他初始化代码...
    
    return 0;
}

static int my_driver_remove(struct platform_device *pdev)
{
    // 通常不需要特殊处理pinctrl
    return 0;
}
```

### 7. 常见应用场景

#### A. GPIO转为其他功能
```c
// 设备树中定义GPIO转I2C的引脚配置
i2c_pins: i2c-pins {
    rockchip,pins = <1 RK_PB0 1 &pcfg_pull_up>,
                  <1 RK_PB1 1 &pcfg_pull_up>;
};
```

#### B. 电源管理状态切换
```c
// 休眠时将引脚设为低功耗状态
static int my_suspend(struct device *dev)
{
    return pinctrl_pm_select_sleep_state(dev);
}

static int my_resume(struct device *dev)
{
    return pinctrl_pm_select_default_state(dev);
}
```

### 8. 学习要点总结

#### 作为外设驱动开发者，重点掌握：
1. **设备树语法**：如何定义pinctrl组和引用
2. **核心API**：`devm_pinctrl_get`, `pinctrl_lookup_state`, `pinctrl_select_state`
3. **使用时机**：在probe函数中何时调用pinctrl API

#### 不需要深究的内容：
1. pinctrl核心层的具体实现
2. 各种内部数据结构的详细字段
3. 寄存器操作的具体实现
4. SoC厂商pinctrl驱动的实现细节

#### 实践建议：
1. 先学会在设备树中定义引脚配置
2. 在现有驱动中添加pinctrl支持
3. 通过实际项目加深理解
4. 遇到问题时查阅具体SoC的pinctrl文档
