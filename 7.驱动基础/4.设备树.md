### 1.什么是设备树
- Device Tree 是一种描述硬件的数据结构，由一系列被命名的节点（node）和属性（property）组成，而节点本身可包含子节点。
- 设备树其实就是把平台总线中的device封装成一个文件，用来描述不同硬件，修改硬件时只需要修改设备树文件不需要重新编译整个驱动
- 设备树文件包括.dts格式和.dtsi格式，.dts格式是有差异的源文件，.dtsi格式是通用的源文件
    - 每个dts文件对应一个arm设备平台，这样更换平台时只需要修改设备树即可
    - dtsi·文件就是多个arm平台通用的部分
    - 添加板级节点时，为了确保不污染其他类型的设备，应当在dts文件添加而不是dtsi文件中添加
- dts、dtsi通过dtc（设备树编译工具）被链接编译成.dtb（设备树二进制文件）

### 2.设备树文件结构
- 设备树从根节点`/`开始，每个设备都是一个节点，节点下可以有属性，属性下可以有值
- 节点之中可以嵌套节点，形成父子关系
- 设备属性通过key-value对（键值对）进行描述，每个属性用分号结束
- 属性描述前加#代表对子节点的语法约束
- 设备树文件可以引入dts/dtsi文件，同C语言引入头文件

**示例**
```c
#include "imx6ull.dtsi"//引入其他dtsi文件
/ {//根节点
    compatible = "arm,vexpress";//根节点的属性1
    model = "Versatile Express";//根节点的属性2
    memory {//子节点1
        reg = <0x00000000 0x80000000>;//子节点1的属性1
        device_type = "memory";//子节点1的属性2
        type = "SDRAM";
        label = "mem";
        ranges = <0x00000000 0x80000000>;
        aliases {//子节点1的子节点1.1
            mem_alias = <0x00000000 0x80000000>;
            mem_virt = <0x00000000 0x80000000>;
            mem_phys = <0x00000000 0x80000000>;
        }
    }
}
```
#### 设备树的数据形式
- 字符串、32位无符号整数、字符串列表、二进制数据
1) 文本字符串（无结束符），可以用双引号表示：
    - a-string-property = "A string";
2) 32 位无符号整数用尖括号限定：
    - cell-property = <0xbeef 123 0xabcd1234>；
3) 二进制数据用方括号限定：
    - binary-property = [0x01 0x23 0x45 0x67];
4) 不同表示形式的数据可以用逗号连在一起：
    - mixed-property = "a string", [0x01 0x23 0x45 0x67], <0x12345678>;
5) 逗号也可以用于创建字符串列表：
    - string-list = "red fish", "blue fish";

### 3.节点名称的规则
格式：`名称@设备地址`
- 节点的名称也不是任意起的，一般要体现设备的类型而不是特点的型号
    - 比如网口，应该命名为ethernet，而不是随意起一个，比如 111
- 设备地址就是用来访问该设备的基地址。但并不是在操作过程中来描述一个地址，而是用来区分设备的
    - 同一级的节点只要地址不一样，名字是可以不唯一的。
    - 设备地址是一个可选选项，可以不写。但为了容易区分和理解，一般是都写的。
**节点标签**
格式：`标签名: 节点名@节点地址`
示例
```c
// uart1就是节点标签 uart就是节点名
//节点标签是为了方便引用，例如&uart1就可以直接引用uart@101f2000这个节点
uart1: uart@101f2000
{
    compatible = "arm,pl011";
    reg = <0x101f2000 0x1000>;
};
```
**节点别名**
格式：`aliases { 别名 = &标签名; }`‘
示例：aliases { uart0 = &uart1; }

**节点引用**
修改节点的时候，不需要找到该节点的位置，只需要引用该节点即可修改
编译时编译器会自动合
格式：`&标签名`，注意不是节点名

### 4.标准属性
1. reg属性
- 用来描述设备的地址范围,表示节点地址资源
- 比如常见的就是寄存器的起始地址及大小
格式：`reg = <地址1 长度 地址2 长度>`
示例
```c
serial@02020000 {
    reg = <0x02020000 0x4000>;
};
```

2. "#address-cells"和"#size-cells"属性
- 用来描述节点地址资源所占的位数，也就是描述子节点reg属性所占的位数
- 其键值是32位无符号整数
- #address-cells设置子节点中reg地址的数量，决定了子节点reg属性中地址信息所占用的字长(32位)
- #size-cells设置子节点中reg地址长度的数量，决定了子节点reg属性中长度信息所占的字长(32 位)
一般reg属性都是和地址有关的内容，和地址相关的信息有两种：起始地址和地址长度，reg 属性的格式一为：
reg = <address1 length1 address2 length2 address3 length3……>
每个“address length”组合表示一个地址范围，其中 address 是起始地址，length 是地址长度，
#address-cells表明address这个数据所占用的字长，#size-cells 表明 length 这个数据所占用
的字长
**示例**
```c
cpu{
    #address-cells = <1>;
    #size-cells = <1>;
    serial@02020000 {
        compatible = "serial";
        reg = <0x02020000 0x4000>;//0x02020000是起始地址，0x4000是长度
    };
};
```
3. compatible属性（兼容性属性）
- 用来描述设备类型，兼容性，用于匹配设备，将设备和驱动绑定起来，是一个字符串列表
- 每个表示一个设备的节点都需要一个compatible属性
例如：compatible = "fsl,imx6ul-evk-wm8960","fsl,imx-audio-wm8960";
```
属性值有两个，分别为“fsl,imx6ul-evk-wm8960”和“fsl,imx-audio-wm8960”，其中“fsl”表示厂商是飞思卡尔，“imx6ul-evk-wm8960”和“imx-audio-wm8960”表示驱动模块名字。所以这个设备首先使用第一个兼容值在 Linux 内核里面查找，看看能不能找到与之匹配的驱动文
件，如果没有找到的话就使用第二个兼容值查。
```


4. status属性
- 用来表示节点的状态，其实就是硬件的状态，用字符串表示
- `okay`表示硬件正常工作
- `disable`表示当前硬件不可用
- `fail`表示因为出错不可用
- `fail-sss`表示某种原因出错不可用，sss 表示具体出错的原因。
- 实际中，基本只用`okay`和`disabl`。

5. range属性
- 用来将子地址空间映射到父地址空间
- 比如一个总线设备节点，其子设备节点的物理地址存在于子总线地址空间，上层的cpu并不认识，因此需要映射到父总线设备节点所处的物理地址空间，从而得到一个全局的、cpu能够识别的地址。
- ranges属性值可以为空或者按照(child-bus-address,parent-bus-address,length)格式编写的数字矩阵
- ranges是一个地址映射/转换表，ranges属性每个项目由子地址、父地址和地址空间长度这三部分组成：
- child-bus-address：子总线地址空间的物理地址，由父节点的#address-cells 确定此物理地址所占用的字长。
- parent-bus-address：父总线地址空间的物理地址，同样由父节点的#address-cells 确定此物理地址所占用的字长。
- length：子地址空间的长度，由父节点的#size-cells 确定此地址长度所占用的字长。
如果ranges属性值为空值，说明子地址空间和父地址空间完全相同，不需要进行地址转换，
对于I.MX6ULL来说，子地址空间和父地址空间完全相同，因此会在 imx6ull.dtsi中找到大量的值为空的 ranges 属性

### 5.查看设备树节点
设备树文件一般在`内核源码/arch/arm/boot/dts` 

设备树在linux系统里面以文件或目录的形式表示出来，这里有两种方法
**方法一**
```bash
cd /proc/device-tree
ls
//此时可以看到根节点的属性列表和子节点
//查看根节点的model属性
cat model
//查看根节点的compatible属性
cat compatible
```
**方法二**
```bash
cd /sys/firmware/devicetree/base/
ls
//同上
```

### 6.编译设备树文件
- 用dtc编译dts、dtsi为dtb文件
1. 安装dtc工具
```bash
apt-get install device-tree-compiler
```
2. 配置环境并编译
```bash
//指定交叉编译器路径
//让make能找到arm-linux-gnueabihf-gcc、arm-linux-gnueabihf-dtc（设备树编译器）等工具
export PATH=/usr/local/arm/gcc-linaro-4.9.4-2017.01-x86_64_arm-linux-gnueabihf/bin:$PATH
//编译设备树文件
//dtbs指的是编译所以设备树文件
//改成xxx.dts可以指定只编译某个设备树文件
make ARCH=arm CROSS_COMPILE=arm-linux-gnueabihf- dtbs
```
3.也可以直接用内核源码的dtc可执行文件编译
```bash
dtc -I dts -O dtb -o imx6ull-evk.dtb imx6ull-evk.dts # 输入dts文件，输出dtb文件
# 反编译dtb文件
dtc -I dtb -O dts -o imx6ull-evk.dts imx6ull-evk.dtb # 输入dtb文件，输出dts文件
```
**前提条件**
1. 已下载 Linux 内核源码：必须在 内核源码根目录执行
2. 已安装交叉编译工具链且配置路径正确
3. 已配置并编译过内核，生成 .config 文件


### 设备树中的of函数
设备树描述了设备的详细信息，这些信息包括数字类型的、字符串类型的、数组类型的，
我们在编写驱动的时候需要获取到这些信息。比如设备树使用reg属性描述了某个外设的寄存
器地址为 0X02005482，长度为 0X400，我们在编写驱动的时候需要获取到reg属性的0X02005482和0X400这两个值，然后初始化外设。

Linux 内核给我们提供了一系列的函数来获取设备树中的节点或者属性信息，这一系列的函数都有一个统一的前缀“of_”，所以也被叫做OF函数设备都是以节点的形式“挂”到设备树上的，因此要想获取这个设备的其他属性信息，
必须先获取到这个设备的节点。Linux 内核使用 device_node 结构体来描述一个节点，
此结构体定义在文件`include "/linux/of.h"`

**描述节点信息的结构体**
```c
struct device_node
{
const char *name; /* 节点名字 */
const char *type; /* 设备类型 */
phandle phandle;
const char *full_name; /* 节点全名 */
struct fwnode_handle fwnode;
struct property *properties; /* 属性 */
struct property *deadprops; /* removed 属性 */
struct device_node *parent; /* 父节点 */
struct device_node *child; /* 子节点 */
// 省略...
// 省略...
}
```
**描述节点属性的结构体**
```c
struct property
{
char *name; /* 属性名字 */
int length; /* 属性长度 */
void *value; /* 属性值 */
struct property *next; /* 下一个属性 */
// 省略...
};
```
#### 获取子节点的of函数
##### 1.查找节点
**通过路径查找指定的节点**
```c
inline struct device_node *of_find_node_by_path(const char *path)
```
**参数说明**
path：节点全路径，如“/backlight”代表根节点的子节点backlight，可以使用节点的别名
**返回值** 成功：返回找到的节点，失败返回 NULL

**通过节点名字查找指定的节点**
```c
struct device_node *of_find_node_by_name(struct device_node *from,const char *name);
```
**参数说明**
from：要搜索的起始节点，NULL表示从根节点开始搜索
name：要查找的节点名字。
**返回值** 成功：返回找到的节点，失败返回 NULL

**通过device_type属性查找指定的节点**
```c
struct device_node *of_find_node_by_type(struct device_node *from,const char *type);
```
**参数说明**
from：要搜索的起始节点，NULL表示从根节点开始搜索
type：要查找的节点类型。
**返回值** 成功：返回找到的节点，失败返回 NULL

**通过compatible属性查找指定的节点**
```c
struct device_node *of_find_node_by_compatible(struct device_node *from,const char *compatible);
```
**参数说明**
from：要搜索的起始节点，NULL表示从根节点开始搜索
compatible：要查找的节点兼容字符串。
**返回值** 成功：返回找到的节点，失败返回 NULL

**通过of_device_id匹配表来查找指定的节点**
```c
struct device_node *of_find_matching_node_and_match(struct device_node *from,const struct of_device_id *matches,const struct of_device_id **match)
```
**参数说明**
from：要搜索的起始节点，NULL表示从根节点开始搜索
matches：要查找的of_device_id匹配表
match：返回匹配到的of_device_id
**返回值** 成功：返回找到的节点，失败返回 NULL

##### 2.查找父节点
- 用于获取指定节点的父节点
```c
struct device_node *of_get_parent(const struct device_node *node)
```
**参数说明**
node：要查找父节点的
**返回值** 成功：返回找到的节点，失败返回 NULL

##### 3.查找子节点
- 用于获取指定节点的子节点
```c
struct device_node *of_get_next_child(const struct device_node *node struct device_node *prev)
```
**参数说明**
node：要查找子节点的节点
prev：前一个子节点，即从哪个子节点开始迭代查找下一个子节点。设为NULL表示从第一个子节点开始。
**返回值** 成功：返回找到的下一个子节点，失败返回 NULL

#### 获取属性值的of函数
- 共5个
节点的属性信息里面保存了驱动所需要的内容，因此对于属性值的提取非常重要，Linux 内核中使用结构体property表示属性，此结构体同样定义在文件`include/linux/of.h`中，内容如下：
```c
struct property {
char *name; /* 属性名字 */
int length; /* 属性长度 */
void *value; /* 属性值 */
struct property *next; /* 下一个属性 */
unsigned long _flags;
unsigned int unique_id;
struct bin_attribute attr;
};
```
##### 1. of_find_property 函数
- 用于查找指定属性
```c
property *of_find_property(const struct device_node *np,const char *name,int *lenp)
```
**参数说明**
- np：设备节点
- name：属性名字
- lenp：属性值的字节数
- 返回值：找到的属性

##### 2. of_property_read_u8/16/32/64 函数
- 有些属性只有一个整型值，这四个函数就是用于读取这种只有一个整型值的属性，分别用于读取u8、u16、u32、u64类型属性值
```c
int of_property_read_u8(const struct device_node *np,const char *propname,u8 *out_value)
int of_property_read_u16(const struct device_node *np,const char *propname,u16 *out_value)
int of_property_read_u32(const struct device_node *np,const char *propname,u32 *out_value)
int of_property_read_u64(const struct device_node *np,const char *propname,u64 *out_value)
```
**参数说明**
- np：设备节点
- proname：要读取的属性名字
- out_value：读取到的数组值
- 返回值：0，读取成功，负值，读取失败


##### 3. of_property_read_u8/16/32/64_array 函数
- 读取属性中u8/u16/u32/u64类型的数组数据，大多数的reg属性都是数组数据，可以使用这4个函数一次读取出reg属性中的所有数据
```c
int of_property_read_u8_array(const struct device_node *np, const char *propname, u8 *out_values, size_t sz)
int of_property_read_u16_array(const struct device_node *np, const char *propname, u16 *out_values, size_t sz)
int of_property_read_u32_array(const struct device_node *np, const char *propname, u32 *out_values, size_t sz)
int of_property_read_u64_array(const struct device_node *np, const char *propname, u64 *out_values, size_t sz)
```
**参数说明**
- np：设备节点
- proname：要读取的属性名字
- out_value：读取到的数组值，分别为 u8、u16、u32 和 u64。
- sz：要读取的数组元素数量
- 返回值：0，读取成功，负值，读取失败

##### 4. of_property_read_string 函数
```c
int of_property_read_string(struct device_node *np,const char *propname,const char **out_string)
```
**参数说明**
- np：设备节点
- proname：要读取的属性名字
- out_string：读取到的字符串值
- 返回值：0，读取成功，负值，读取失败
- 功能：of_property_read_string 函数用于读取属性中字符串值

#### 5. of_iomap 函数
- 用于直接内存映射
- 可以代替ioremap
```c
void __iomem *of_iomap(struct device_node *np,int index)
```
**参数说明**
- np：设备节点
- index：reg 属性中要完成内存映射的段，如果 reg 属性只有一段的话 index 就设置 0。
- 返回值：经过内存映射后的虚拟内存首地址，如果为 NULL 的话表示内存映射失败


#### 6.of_property_count_elems_of_size函数
- 获取属性中指定大小的元素数量
- 例如可以获取reg数组的大小
```c
int of_property_count_elems_of_size(const struct device_node *np, const char *propname, size_t elem_size)
```
**参数说明**
- np：设备节点
- propname：要读取的属性名字
- elem_size：元素大小，即每个元素占用的字节数
- 返回值：属性中指定大小的元素数量

#### 7.of_n_addr_cells函数
- 获取设备节点的n-addr-cells属性值
```c
int of_n_addr_cells(struct device_node *np)
```
**参数说明**
- np：设备节点
- 返回值：n-addr-cells属性值

#### 8.of_n_size_cells函数
- 获取设备节点的n-size-cells属性值
```c
int of_n_size_cells(struct device_node *np)
```
**参数说明**
- np：设备节点
- 返回值：n-size-cells属性值

#### 9.of_get_address函数
- 获取设备节点的reg属性值
```c
int of_get_address(struct device_node *np, int index, u64 *addr, u32 *size, u32 *flags)
```
**参数说明**
- np：设备节点
- index：reg 属性中要获取的地址段索引
- addr：输出参数，用于存储获取到的地址值
- size：输出参数，用于存储获取到的大小值
- flags：输出参数，用于存储获取到的标志位值
- 返回值：0，获取成功，负值，获取失败

#### 10.of_translate_address函数
- 用于将设备树中的地址转换为物理地址
```c
int of_translate_address(struct device_node *np, const struct property *prop, const u32 *addr, u64 *out_addr, u32 *out_size, u32 *out_flags)
```
**参数说明**
- np：设备节点
- prop：reg 属性
- addr：要转换的地址值
- out_addr：输出参数，用于存储转换后的物理地址值
- out_size：输出参数，用于存储转换后的大小值
- out_flags：输出参数，用于存储转换后的标志位值
- 返回值：0，转换成功，负值，转换失败

#### 11.of_address_to_resource函数
- 用于将设备树中的地址转换为资源结构体
```c
int of_address_to_resource(struct device_node *np, const struct property *prop, const u32 *addr, struct resource *r)
```
**参数说明**
- np：设备节点
- prop：reg 属性
- addr：要转换的地址值
- r：输出参数，用于存储转换后的资源结构体
- 返回值：0，转换成功，负值，转换失败


**示例**
```c

#include <linux/init.h> //初始化头文件
#include <linux/module.h> //最基本的文件，支持动态添加和卸载模块。
#include <linux/of.h> //添加of函数所在的头文件
```

### 设备树描述中断控制器
**常见属性**
- compatible：兼容属性，表示中断控制器的兼容字符串，例如：intel,apic，用于匹配中断控制器的驱动
- interrupt-controller：表示中断控制器，用于描述中断控制器的中断源
- interrupt-cells：表示中断控制器中断源的属性数量，用于描述中断控制器中断源的属性数量，例如：1，表示每个中断源有一个属性
- interrupt-parent：表示中断控制器的父节点，例如：/soc/interrupt-controller@fed00000

**示例**
```c
// GPIO1 控制器节点定义
gpio1: gpio@0209c000 {
    compatible = "fsl,imx6ul-gpio", "fsl,imx35-gpio"; // 兼容属性
    interrupt-controller;
    // 寄存器地址范围：起始地址 0x0209c000，长度 0x40000
    reg = <0x0209c000 0x40000>;

    // 中断配置列表
    // 1. GIC SPI 中断 66：高电平触发
    // 2. GIC SPI 中断 67：高电平触发
    interrupts = <GIC_SPI 66 IRQ_TYPE_LEVEL_HIGH>,
                 <GIC_SPI 67 IRQ_TYPE_LEVEL_HIGH>;

    // 声明该节点为 GPIO 控制器
    gpio-controller;
    
    // GPIO 单元数量 = <2>
    // 引用格式: <&gpio1 引脚号 标志位>
    #gpio-cells = <2>;

    // 声明该节点为中断控制器
    interrupt-controller;
    
    // 中断单元数量 = <2>
    // 引用格式: <&gpio1 中断号 触发方式>
    #interrupt-cells = <2>;
};
```
```c
//触摸屏设备节点
edt-ft5x06@38 {
    // 兼容属性：匹配多种驱动
    // 包含: edt,ft5306 / edt,ft5x06 / edt,ft5406
    compatible = "edt,edt-ft5306", "edt,edt-ft5x06", "edt,edt-ft5406";
    // 引脚控制名称配置，对应引脚控制 phandle
    pinctrl-names = "default";
    pinctrl-0 = <&ts_int_pin>, <&ts_reset_pin>;
    reg = <0x38>;//
    // 中断父控制器：指定使用 gpio1
    interrupt-parent = <&gpio1>;//不存在时默认使用父节点的中断控制器
    // 中断配置：GPIO1的9号引脚，低电平触发
    // 注意：此处中断触发类型与上方 GPIO 配置的高电平不同
    interrupts = <9 IRQ_TYPE_LEVEL_LOW>;
    // 中断 GPIO 配置：GPIO1 引脚 9，低电平有效
    // 作用与interrupts类似，均用于定义中断引脚
    irq-gpios = <&gpio1 9 GPIO_ACTIVE_LOW>;
};
```
**解释**
1. **`interrupts` 里写什么，完全由 `interrupt-parent` 决定**
2. **`interrupt-parent` 指向谁，就看那个节点的 `#interrupt-cells = <x>`**
- `#interrupt-cells = <1>` → 中断号 1 个数字
- `#interrupt-cells = <2>` → 中断号 + 触发方式
- `#interrupt-cells = <3>` → 很少见，ARM GIC 专用


1. 最常见 → 接在 GPIO 引脚上的中断（上述例子就是这种）
- 第一步：看 GPIO 控制器怎么定义
```c
gpio1: gpio@0209c000 {
    compatible = "fsl,imx6ul-gpio";
    reg = <0x0209c000 0x40000>;
    interrupts = <GIC_SPI 66 IRQ_TYPE_LEVEL_HIGH>,
                 <GIC_SPI 67 IRQ_TYPE_LEVEL_HIGH>;

    interrupt-controller;       // 它是中断控制器
    #interrupt-cells = <2>;     ✅ 规定：需要 2 个参数
};
```

`#interrupt-cells = <2>`
→ 格式固定为：
```
interrupts = <引脚编号 触发方式>;
```

- 第二步：你的设备这样写
```c
ft5x06@38 {
    interrupt-parent = <&gpio1>;   // 中断来自 gpio1
    interrupts = <9 IRQ_TYPE_LEVEL_LOW>;
};
```
**使用 GPIO1_IO09 引脚作为中断，低电平触发。**

2. 直接接到 CPU 主中断控制器 GIC（ARM 芯片都有）
GIC 定义一般是：
```c
gic: interrupt-controller {
    compatible = "arm,gic-400";
    interrupt-controller;
    #interrupt-cells = <3>;   ✅ 3 个参数
};
```

`#interrupt-cells = <3>`
→ 格式：
```
interrupts = <类型 中断号 触发方式>;
```

设备这样写：
```c
uart@02020000 {
    interrupt-parent = <&gic>;
    interrupts = <GIC_SPI 20 IRQ_TYPE_LEVEL_HIGH>;
};
```
**使用 GIC 全局中断号 20，高电平触发**


3. 什么都不写，继承父节点（最容易懵的点）
假如 I2C 节点已经指定了 gpio1：
```c
i2c@30a20000 {
    interrupt-parent = <&gpio1>;  // 父节点写了

    ft5x06@38 {
        interrupts = <9 IRQ_TYPE_LEVEL_LOW>;
        // 这里不写 interrupt-parent 也可以
    };
};
```
**子节点会自动继承父节点的 interrupt-parent**

### 设备树描述时钟
- 分为生产者和消费者

#### 生产者
**属性**
- clock-cells：时钟输出路数
- clock-name：时钟输出名称列表
- clock-frequency：时钟频率
```c
osc24m: osc24m{
    compatible = "clock";
    clock-frequency = <24000000>;
    clock-output-names = "osc24m";
    #clock-cells = <0>;
};
```
- assigned-clocks：分配给消费者的时钟列表
- assigned-clock-rates：分配给消费者的时钟频率列表
```c
cru: clock-controller@fdd20000 {
    #clock-cells = <1>;
    assigned-clocks = <&pmucru CLK_RTC_32K>, <&cru ACLK_RKVDEC_PRE>;
    assigned-clock-rates = <32768>, <30000000>;
};
```
- clock-indices：时钟索引列表,不提供则默认是0,1,2……，用来指定clock-output-names的索引
```c
scpi_clk: clocks-1 {
    #clock-cells = <1>;
    clock-indices = <3>;
    clock-output-names = "pxlclk";
};
```

#### 消费者
**属性**
- clocks：指定时钟源
- clock-names：时钟消费者列表
```c
clock:clock{
    clocks = <&cru CLK_VOP>;
    clock-names = "clk_vop";
};
```
